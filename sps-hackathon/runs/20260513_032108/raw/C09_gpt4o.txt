Specification for Checkout Flow:

Core Behavior:
- The system must facilitate a seamless checkout process for users purchasing items from an online store.
- The checkout flow should guide users from the shopping cart to order confirmation with minimal friction.
- The system should be optimized to maximize conversion rates, ensuring users complete their purchases.

Inputs:
- User shopping cart data, including selected items, quantities, and prices.
- User account information, if logged in (e.g., name, email, shipping address).
- Payment information (e.g., credit card details, PayPal account).
- Discount codes or promotional offers, if applicable.

Outputs:
- Order confirmation page displaying order details, including items purchased, total cost, and estimated delivery date.
- Confirmation email sent to the user with order details and receipt.
- Transaction data sent to the payment gateway for processing.
- Inventory update reflecting the purchased items.

Assumptions:
- The checkout flow will consist of the following steps: Review Cart, Enter Shipping Information, Enter Payment Information, Review Order, and Confirmation.
- Users can check out as guests or logged-in users.
- The system will support multiple payment methods, including credit/debit cards and PayPal.
- The system will apply any valid discount codes automatically during checkout.
- The checkout process will be mobile-responsive and accessible on various devices.

Failure Modes:
- If payment processing fails, the system will display an error message and allow the user to retry or choose a different payment method.
- If inventory is insufficient for any item in the cart, the system will notify the user and suggest alternatives or allow them to adjust quantities.
- If there is a system error during checkout, the user will be redirected to a support page with contact information for assistance.

Edge Cases:
- Users attempting to check out with an empty cart will be redirected to the shopping page with a message prompting them to add items.
- If a user enters invalid shipping or payment information, the system will highlight the errors and prompt for corrections.
- The system will handle timeouts or connectivity issues by saving the user's progress and allowing them to resume the checkout process later.

Security Constraints:
- All sensitive data, such as payment information, must be encrypted during transmission and storage.
- The system must comply with PCI DSS standards for handling credit card information.
- User authentication must be required for accessing saved payment methods and addresses.

Performance Constraints:
- The checkout process should complete within 3 seconds per page load under normal conditions.
- The system should handle peak loads, such as during sales events, without significant degradation in performance.
- The system should be scalable to accommodate increased traffic and transaction volumes.
