Specification for Password Reset Flow

Core Behavior:
- The system must provide a secure mechanism for users to reset their passwords when they have forgotten them or wish to change them for security reasons.
- The system must verify the identity of the user requesting the password reset.
- The system must allow the user to set a new password and confirm it.
- The system must update the user's password in the database upon successful completion of the reset process.

Inputs:
- User's email address or username to initiate the password reset process.
- Verification token sent to the user's registered email address.
- New password entered by the user.
- Confirmation of the new password entered by the user.

Outputs:
- A confirmation message indicating that a password reset email has been sent.
- A notification email containing a secure link or token for password reset.
- A success message upon successful password reset.
- An error message if the password reset process fails at any stage.

Assumptions:
- The system uses email as the primary method for user verification during the password reset process.
- The system assumes that the email address provided by the user is valid and registered in the system.
- The system assumes that the user has access to the email account associated with their account.
- The system assumes that the password reset link or token is valid for a limited time (e.g., 24 hours).
- The system assumes that the new password must meet predefined security criteria (e.g., minimum length, complexity).

Failure Modes:
- If the email address or username provided does not exist in the system, the system should return a generic error message to avoid revealing account existence.
- If the verification token is invalid or expired, the system should return an error message and prompt the user to request a new password reset.
- If the new password and confirmation password do not match, the system should return an error message and prompt the user to re-enter the passwords.
- If the new password does not meet security criteria, the system should return an error message specifying the criteria that were not met.

Edge Cases:
- The user enters an email address or username with leading or trailing spaces.
- The user attempts to use the password reset link multiple times.
- The user requests multiple password reset emails in a short period.
- The user attempts to reset the password using an expired or previously used token.

Security Constraints:
- The password reset link or token must be securely generated and contain sufficient entropy to prevent guessing attacks.
- The system must use HTTPS to encrypt all communications related to the password reset process.
- The system must not disclose whether an email address or username exists in the system when initiating the password reset.
- The system must log all password reset attempts for auditing and monitoring purposes.

Performance Constraints:
- The system must send the password reset email within a reasonable time frame (e.g., within 5 minutes of the request).
- The system must handle concurrent password reset requests efficiently without degrading performance for other users.
